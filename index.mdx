CC Switch gives you a single desktop interface to manage all your AI coding tools. Instead of editing JSON, TOML, and `.env` files by hand, you get one-click provider switching, 50+ built-in presets, unified MCP and Skills management, and a local proxy with automatic failover — all backed by a reliable SQLite database.

## What CC Switch does

CC Switch solves a real pain point for developers who use multiple AI coding tools: each tool (Claude Code, Codex, Gemini CLI, OpenCode, OpenClaw) stores its configuration in a different format and location. Switching providers means manually editing files — and there's no unified way to share MCP servers or Skills across tools.

## Get up and running

<Steps>
  <Step title="Download and install CC Switch">
    Download the installer for your platform from the [GitHub Releases](https://github.com/farion1231/cc-switch/releases) page. See [Installation](/installation) for platform-specific instructions.
  </Step>
  <Step title="Add a provider">
    Click **+** in the top-right corner, pick a preset (50+ available), enter your API key, and click **Add**. See [Add Provider](/providers/add-provider).
  </Step>
  <Step title="Switch and activate">
    Click **Enable** on a provider card in the main UI, or use the system tray menu for instant switching. Restart your terminal (except Claude Code, which hot-reloads). See [Switch Provider](/providers/switch-provider).
  </Step>
  <Step title="Extend with MCP, Skills, and Prompts">
    Use the **MCP**, **Skills**, and **Prompts** panels to extend your AI tools with servers, capabilities, and system prompts — all synced across apps. See [Extensions](/extensions/mcp-servers).
  </Step>
</Steps>

<Note>
  CC Switch stores all data locally in `~/.cc-switch/cc-switch.db` (SQLite). Your API keys and configurations never leave your machine unless you explicitly enable cloud sync.
</Note>
