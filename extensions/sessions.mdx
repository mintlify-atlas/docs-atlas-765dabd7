title: 'Browse and restore conversation history across tools'

Session Manager gives you a single place to browse all conversation history from every AI coding tool you use. Instead of hunting through scattered cache directories, you can search across sessions from all five supported apps, read the full transcript, and resume any conversation with one click.

## Opening Session Manager

Click the **Sessions** button in the main navigation bar toolbar. The Sessions button is visible regardless of which app is currently selected.

## Interface overview

Session Manager uses a two-column layout:

- **Left panel** — session list with search bar, provider filter, and toolbar actions.
- **Right panel** — full details for the selected session: title, last active timestamp, project directory, resume command, and complete conversation transcript.

Session entries display the provider icon, session title, and relative last-active time (e.g., "5 min ago"). Sessions are sorted newest first.

## Finding a session

<Steps>
  <Step title="Open Session Manager">
    Click **Sessions** in the top navigation bar.
  </Step>
  <Step title="Filter by provider (optional)">
    Click the provider filter dropdown in the top-right of the left panel and select the app you want — Claude Code, Codex, OpenCode, OpenClaw, or Gemini CLI. Select **All** to show every session.
  </Step>
  <Step title="Search for the session">
    Type in the search box to filter by session ID, title, summary, project directory, or source file path. Results update in real time. Press **Esc** to clear the search.
  </Step>
  <Step title="Select the session">
    Click a session entry in the list. The right panel shows the full transcript with messages color-coded by role: green for user, blue for AI/assistant, amber for system, and purple for tool messages.
  </Step>
  <Step title="Resume or act on the session">
    Click **Resume** (play icon) to continue the conversation in your terminal, or **Delete** (trash icon) to permanently remove it.
  </Step>
</Steps>

## Resuming a session

Click the **Resume** button on a selected session to reopen it in the CLI tool.

**On macOS**, CC Switch launches your preferred terminal application with the resume command and opens it in the session's project directory. Supported terminals include Terminal.app, iTerm2, Ghostty, Kitty, WezTerm, and Alacritty. If the terminal launch fails, the command is copied to your clipboard instead.

**On other platforms**, the resume command is copied to your clipboard. Paste it into your terminal to continue.

The Resume button is disabled if the session has no resume command available.

### Directory picker for Claude sessions

Starting in v3.13.0, resuming a Claude Code session shows a **directory picker** before launch. This lets you override the default project directory — useful when the original project has been moved, renamed, or is no longer accessible.

1. Click **Resume** on a Claude Code session.
2. In the directory picker, confirm the default path or choose a new one.
3. CC Switch launches the terminal in the selected directory.

<Note>
The directory picker is currently available for Claude Code sessions only. Codex, Gemini CLI, OpenCode, and OpenClaw sessions resume in the session's original project directory without a picker.
</Note>

## Deleting sessions

Click the **Delete** button (trash icon) on a selected session. A confirmation dialog is shown before the session file is permanently removed. Sessions without a local source file path cannot be deleted.

### Batch delete

1. Click the **Batch Mode** button (checkbox icon) in the left panel toolbar.
2. Select sessions using the checkboxes, or click **Select All** to select every filtered result.
3. Click **Batch Delete** (red trash icon) and confirm. The dialog shows how many sessions will be deleted, and results report the number of successful deletions and any failures.

## Session sources

CC Switch reads sessions from the following locations per app:

| App | Session location |
|---|---|
| Claude Code | `~/.cache/claude/projects/*.jsonl` |
| Codex | Codex config sessions directory |
| OpenCode | `~/.local/share/opencode/` (JSON files or SQLite) |
| OpenClaw | `~/.openclaw/agents/<agent>/sessions/*.jsonl` |
| Gemini CLI | `~/.cache/gemini/tmp/<project_hash>/chats/` |

OpenCode sessions may come from both JSON files and an SQLite database — CC Switch automatically deduplicates them.

## Workspace editor (OpenClaw)

When OpenClaw is the selected app, CC Switch also provides a **Workspace** panel for editing OpenClaw's configuration files. Click **Workspace** in the navigation bar to open it.

All workspace files are stored in `~/.openclaw/workspace/`. CC Switch manages nine files:

| File | Purpose |
|---|---|
| `AGENTS.md` | Agent definitions and configuration |
| `SOUL.md` | System soul and personality settings |
| `USER.md` | User profile information |
| `IDENTITY.md` | Identity and role definition |
| `TOOLS.md` | Available tools configuration |
| `MEMORY.md` | System memory |
| `HEARTBEAT.md` | Heartbeat configuration |
| `BOOTSTRAP.md` | Bootstrap sequence |
| `BOOT.md` | Boot configuration |

Click any file card to open it in the Markdown editor with live preview. Click **Save** to write changes to disk. Files that do not yet exist are created on first save.

### Daily Memory

The Workspace panel also includes a **Daily Memory** journal stored in `~/.openclaw/workspace/memory/`. Click the **Daily Memory** card to open it.

- Entries are sorted by date, newest first, with a file size and two-line preview for each.
- Click **Create Today** to open a new note for today's date (`YYYY-MM-DD.md`). If today's note already exists, it opens for editing.
- Press **Cmd/Ctrl+F** or click the search icon to search across all memory files. Results show match count, a snippet preview, and file metadata.
- Hover over any entry and click the delete icon to remove it. Deletion cannot be undone.
