title: "Get up and running with CC Switch in 5 minutes"

---

This guide walks you through the essential workflow: installing CC Switch, adding a provider, switching to it, and verifying the configuration is active. By the end you'll have a working provider set up for at least one AI CLI tool.

## The 5-step workflow

<Steps>
  <Step title="Install CC Switch">
    Download and install CC Switch for your platform. See [Installation](/installation) for platform-specific instructions, or use the quick commands below:

    ```bash
    # macOS via Homebrew
    brew tap farion1231/ccswitch && brew install --cask cc-switch
    ```

    ```bash
    # Arch Linux via paru
    paru -S cc-switch-bin
    ```

    For Windows (MSI) and Linux (DEB/RPM/AppImage), download the latest package from the [GitHub Releases](https://github.com/farion1231/cc-switch/releases) page.
  </Step>
  <Step title="Add a provider">
    Click the **+** button in the top-right corner of the main interface.

    1. Open the **Preset** dropdown and select your provider. Common presets include DeepSeek, MiniMax, Kimi, PackyCode, and many others. Select **Custom** to enter a base URL manually.
    2. Enter your **API key** in the API Key field. For preset providers, the endpoint URL is filled in automatically.
    3. Click **Add**.

    <Tip>
      Presets automatically populate the endpoint URL — you only need to provide your API key.
    </Tip>
  </Step>
  <Step title="Switch to the new provider">
    The provider you added now appears as a card in the main list. You have two ways to switch:

    **From the main interface:** Click the **Enable** button on the provider card. The card border turns blue to indicate it is now active.

    **From the system tray:** Right-click the CC Switch icon in the system tray, find the app submenu (for example, "Claude"), and click the provider name directly.
  </Step>
  <Step title="Activate the configuration">
    After switching, each CLI tool picks up the new configuration differently:

    | Application | Activation method |
    |---|---|
    | **Claude Code** | Instant — supports hot-reload, no restart needed |
    | **Gemini CLI** | Instant — re-reads config on each request |
    | **Codex** | Close and reopen your terminal |
    | **OpenCode** | Close and reopen your terminal |
    | **OpenClaw** | Close and reopen your terminal |

    For tools that require a terminal restart, close all open terminal sessions running that tool, then open a new terminal window.
  </Step>
  <Step title="Verify the configuration">
    Launch the CLI tool and send a test message to confirm it is using the new provider:

    ```bash
    # Claude Code
    claude
    ```

    ```bash
    # Codex
    codex
    ```

    ```bash
    # Gemini CLI
    gemini
    ```

    ```bash
    # OpenCode
    opencode
    ```

    ```bash
    # OpenClaw
    openclaw
    ```

    Once the tool is running, type a simple message like `Hello, briefly introduce yourself`. If the AI responds normally, the provider switch was successful.
  </Step>
</Steps>

## First-launch note for Claude Code

If Claude Code shows a login prompt or onboarding wizard the first time you run it after switching, you can skip this flow using the **Skip Claude Code first-run confirmation** setting:

1. Open CC Switch and go to **Settings > General**.
2. Enable the **Skip Claude Code first-run confirmation** toggle.
3. Restart Claude Code.

<Note>
  This setting writes the `skipIntroduction` field to `~/.claude/settings.json`, which tells Claude Code to skip its official onboarding flow. Use it if you are already familiar with Claude Code and do not need the walkthrough.
</Note>

## Troubleshooting

**Changes not taking effect after switching?**
Make sure you restarted the terminal or CLI tool for tools that require it (Codex, OpenCode, OpenClaw). The configuration file is updated at switch time, but running processes do not automatically reload it.

**Can't find your provider in the preset list?**
Select **Custom** in the preset dropdown and enter the base URL manually. See [Add Provider](/providers/add-provider) for the full configuration reference.

**Need to switch back to official login?**
Add the **Official Login** preset (for Claude or Codex) or the **Google Official** preset (for Gemini), enable it, and follow the tool's normal login flow.
