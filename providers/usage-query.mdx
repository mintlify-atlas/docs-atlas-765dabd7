CC Switch can display remaining quota or account balance directly on each provider card. The display system is split into two categories: **Auto Query**, which works out of the box for official subscriptions, and **Manual Enable**, which covers third-party providers and requires a one-time configuration step.

| Category | Covered providers | User action required |
|----------|-------------------|----------------------|
| **Auto Query** | Claude / Codex / Gemini official subscriptions, GitHub Copilot, Codex OAuth reverse proxy | None — enabled by default |
| **Manual Enable (built-in templates)** | Token Plan (Kimi, Zhipu GLM, MiniMax), third-party balances (DeepSeek, StepFun, SiliconFlow, OpenRouter, Novita AI) | Turn on the Usage Query toggle and select a template |
| **Manual Enable (custom script)** | Any provider not covered by built-in templates | Write a JavaScript extractor script |

---

## Auto Query — official subscriptions (v3.13.0+)

For the three provider types below, quota information appears automatically at the bottom of the provider card once the provider is activated. No additional setup is needed.

| Type | Examples | Displayed information |
|------|----------|-----------------------|
| Official subscriptions | Claude / Codex / Gemini official login | Official subscription quota |
| GitHub Copilot | Copilot provider card | Premium interactions remaining |
| Codex OAuth | Codex OAuth reverse proxy (Claude provider) | ChatGPT account Codex quota |

### How it looks

The card footer shows:
- **Usage percentage** — colored green (< 70%), orange (70–89%), or red (≥ 90%)
- **Reset countdown** — e.g., `7d12h until reset`
- A **refresh icon** to manually re-query at any time

<Note>
  For these three auto-query types, the Health Check and Usage Query Config buttons are hidden on the card to avoid interfering with the built-in display.
</Note>

If a token fails to refresh, the card shows a yellow **Session Expired** warning. Go to **Settings → OAuth Auth Center**, remove the account, and log in again.

## Manual Enable — built-in templates

All other providers require you to manually turn on the Usage Query feature. The same vendor endpoint often supports multiple query modes (plan quota vs. account balance), so CC Switch cannot automatically infer which one you want — the built-in query is disabled by default.

### Providers with ready-to-use templates

| Category | Providers |
|----------|-----------|
| Token Plan (plan quota with progress bar) | Kimi, Zhipu GLM, MiniMax |
| Third-party balance | DeepSeek, StepFun, SiliconFlow, OpenRouter, Novita AI |

### Enable steps

<Steps>
  <Step title="Open the Usage Query configuration">
    Hover over the provider card to reveal action buttons, then click the **Usage Query** button (chart icon).
  </Step>
  <Step title="Turn on the toggle">
    At the top of the configuration panel, enable the **Enable Usage Query** switch.
  </Step>
  <Step title="Select a template">
    Choose the appropriate built-in template (Token Plan, third-party balance, or Custom), or select **Custom** to write your own script.
  </Step>
  <Step title="Fill in credentials if needed">
    Most built-in templates reuse the provider's own API key and base URL. Leave these blank to inherit from the provider, or override them here.
  </Step>
  <Step title="Test the script">
    Click **Test Script** to send a real request and verify the extractor returns data correctly.
  </Step>
  <Step title="Save">
    After saving, the quota information will appear at the bottom of the card the next time the provider is in **Currently Active** state.
  </Step>
</Steps>

<Note>
  Background auto-refresh only triggers when the provider is in **Currently Active** state. Set the auto-query interval to `0` to disable automatic background polling.
</Note>

## Auto Query interval

The interval controls how often CC Switch automatically refreshes usage data in the background while a provider is active.

| Setting | Effect |
|---------|--------|
| `0` | Disable auto-refresh (manual refresh only) |
| `1–1440` | Refresh every N minutes (up to 24 hours) |

Set a reasonable interval to avoid excessive API requests that consume quota or trigger rate limits.

## Custom script query (advanced)

When no built-in template covers your provider, you can write a JavaScript extractor script. This is suited for self-hosted proxy services, private deployments, and APIs with non-standard response formats.

### Script structure

A script must return a single object with a `request` configuration and an `extractor` function:

```javascript
({
  request: {
    url: "{{baseUrl}}/user/balance",
    method: "GET",
    headers: {
      "Authorization": "Bearer {{apiKey}}",
      "User-Agent": "cc-switch/1.0"
    }
  },
  extractor: function(response) {
    return {
      isValid: response.is_active || true,
      remaining: response.balance,
      unit: "USD"
    };
  }
})
```

### Variable placeholders

| Placeholder | Value at runtime |
|-------------|-----------------|
| `{{apiKey}}` | The provider's configured API key |
| `{{baseUrl}}` | The provider's configured base URL |
| `{{accessToken}}` | Configured Access Token (New API template) |
| `{{userId}}` | Configured User ID (New API template) |

### Extractor return format

The object returned by the `extractor` function supports the following fields:

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `remaining` | number | Yes | Remaining balance or quota |
| `unit` | string | Yes | Unit label, e.g. `USD`, `CNY`, `times` |
| `isValid` | boolean | No | Whether the account is valid; defaults to `true` |
| `invalidMessage` | string | No | Message to show when `isValid` is `false` |
| `planName` | string | No | Plan name; supports multi-plan display |
| `total` | number | No | Total balance |
| `used` | number | No | Used amount |
| `extra` | object | No | Additional display data |

When `planName` is present, the card shows a multi-plan summary. Clicking it expands individual plan details.

### Preset templates

CC Switch ships three built-in script templates to start from:

<AccordionGroup>
  <Accordion title="Generic template">
    Suitable for most providers with a standard balance endpoint:

    ```javascript
    ({
      request: {
        url: "{{baseUrl}}/user/balance",
        method: "GET",
        headers: {
          "Authorization": "Bearer {{apiKey}}",
          "User-Agent": "cc-switch/1.0"
        }
      },
      extractor: function(response) {
        return {
          isValid: response.is_active || true,
          remaining: response.balance,
          unit: "USD"
        };
      }
    })
    ```
  </Accordion>
  <Accordion title="New API template">
    Designed for self-hosted New API gateway services:

    ```javascript
    ({
      request: {
        url: "{{baseUrl}}/api/user/self",
        method: "GET",
        headers: {
          "Content-Type": "application/json",
          "Authorization": "Bearer {{accessToken}}",
          "New-Api-User": "{{userId}}"
        },
      },
      extractor: function (response) {
        if (response.success && response.data) {
          return {
            planName: response.data.group || "Default Plan",
            remaining: response.data.quota / 500000,
            used: response.data.used_quota / 500000,
            total: (response.data.quota + response.data.used_quota) / 500000,
            unit: "USD",
          };
        }
        return {
          isValid: false,
          invalidMessage: response.message || "Query failed"
        };
      },
    })
    ```

    Additional fields: **Base URL** (New API service URL), **Access Token**, **User ID**.
  </Accordion>
  <Accordion title="Custom template">
    A blank template for full control over request and extraction logic. Use this for APIs with non-standard response structures.
  </Accordion>
</AccordionGroup>

### General configuration options

| Option | Default | Description |
|--------|---------|-------------|
| **Timeout** | 10 seconds | Request timeout in seconds |
| **Auto Query Interval** | 0 (disabled) | Background refresh interval in minutes (0–1440) |

## Endpoint speed test

After adding a provider, you can measure latency to its endpoints to pick the fastest one.

1. Click the **Speed Test** button on the provider card.
2. Add multiple endpoint URLs in the speed test panel.
3. Click **Test** to run latency measurements.
4. Select the endpoint with the lowest result.

**Result color coding:**

| Color | Latency | Quality |
|-------|---------|---------|
| Green | < 500 ms | Excellent |
| Yellow | 500–1000 ms | Fair |
| Red | > 1000 ms | Slow |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Auto Query not showing (official subscription types)">
    1. Confirm the provider is an official subscription type — Claude / Codex / Gemini official login, GitHub Copilot, or Codex OAuth reverse proxy.
    2. The provider must be in **Currently Active** state (inactive providers do not trigger queries).
    3. For OAuth types (Copilot / Codex OAuth), check if the token is still valid. A **Session Expired** message on the card means the token needs to be refreshed — go to **OAuth Auth Center**, remove the account, and log in again.
    4. Verify network access to the official quota endpoint.
  </Accordion>
  <Accordion title="Manual Enable still not showing quota">
    1. Check that the **Enable Usage Query** toggle at the top of the Usage Query panel is on.
    2. Confirm you selected a suitable template (Token Plan / third-party balance / custom).
    3. Click **Test Script** to see the specific error message.
    4. Ensure required fields (API Key, Base URL) are correctly filled.
    5. Remember that background auto-refresh only triggers when the provider is in **Currently Active** state.
  </Accordion>
  <Accordion title="Query failed">
    1. Verify the API key is correct.
    2. Verify the Base URL is correct.
    3. Check network connectivity to the provider endpoint.
    4. Increase the timeout if the provider is slow to respond.
  </Accordion>
  <Accordion title="Empty response or extractor returns nothing">
    1. Check that the extractor function has a `return` statement.
    2. Verify the response data structure matches what the extractor expects.
    3. Use **Test Script** to view the raw API response and adjust accordingly.
  </Accordion>
</AccordionGroup>

<Note>
  Usage queries consume a small amount of the provider's request quota. Choose an auto-query interval appropriate for your plan to avoid unnecessary consumption. API keys and access tokens are stored locally and are never uploaded.
</Note>
